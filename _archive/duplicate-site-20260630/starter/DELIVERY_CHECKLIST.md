# LP納品・公開チェックリスト

このテンプレートは、画像・色・文章だけを変更して即公開するものではありません。以下を案件責任者が確認してから公開してください。

## 必須

- [ ] ブランド名、会社名、所在地、代表者、連絡先を実値へ変更
- [ ] `contact@example.com` を実在する問い合わせ先またはフォームへ変更
- [ ] 架空の実績数値・事例を削除し、根拠と顧客承諾が確認できる内容だけ掲載
- [ ] 画像・ロゴ・フォントの商用利用権、人物の肖像利用許諾を確認
- [ ] サンプル実績・事例の注意書きを削除する前に、数値根拠と顧客掲載許諾を保存
- [ ] プライバシーポリシーを実際の取得情報・利用目的・委託先に合わせて確定
- [ ] 販売・有料申込みがある場合、取引形態に応じた特定商取引法表示を確認
- [ ] 公開URL、title、description、OGP画像、faviconを設定
- [ ] canonical URL、OGP画像の絶対URL、SNS共有表示を確認
- [ ] 公開直前に全ページの `noindex, nofollow` を削除
- [ ] HTTPS、独自ドメイン、404ページ、リダイレクトを確認
- [ ] PC・スマホ実機、主要ブラウザ、キーボード操作を確認
- [ ] 全リンク、問い合わせ送信、返信メール、迷惑メール判定を確認
- [ ] フォームの入力検証、スパム対策、送信制限、保存期間、削除手順を確認
- [ ] `npm run preflight` が成功するまで仮情報・公開禁止設定を解消

## 計測・Cookie

- [ ] GA4、広告タグ、ヒートマップ等の利用有無を確定
- [ ] 利用する場合、プライバシーポリシー・Cookie同意の要否を確認
- [ ] コンバージョンイベントとテスト除外を確認

## 性能・品質

- [ ] 画像をWebP/AVIF等へ最適化し、適切な寸法で出力
- [ ] Lighthouse/PageSpeedでモバイル性能・アクセシビリティ・SEOを確認
- [ ] HTML/CSS/JSのビルドが成功することを確認
- [ ] `npm run build` と `npm run preflight` を実行
- [ ] 依存関係の脆弱性監査を実行し、対応方針を記録
- [ ] 配信環境でCSP、HSTS、Referrer-Policy等のセキュリティヘッダーを確認
- [ ] 外部フォント・外部サービス障害時も内容を読めることを確認

## 運用・引継ぎ

- [ ] ドメイン、サーバー、フォーム、計測ツールの管理者と更新担当を明確化
- [ ] ソース、画像原本、ライセンス証跡、公開手順、ロールバック手順を引き渡し
- [ ] 問い合わせ通知、障害検知、定期的な内容更新の運用を確認

## 法務確認

法的要件は事業内容・取得情報・契約方法・対象顧客により異なります。最終判断は依頼主の法務担当者・専門家が行ってください。

- 個人情報保護委員会「法令・ガイドライン等」: https://www.ppc.go.jp/personalinfo/legal/
- 消費者庁「特定商取引法ガイド 通信販売」: https://www.no-trouble.caa.go.jp/what/mailorder/
